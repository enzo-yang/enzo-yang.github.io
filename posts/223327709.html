又双叒叕开始一个新博客

写作冲动总是一阵阵的来，距离上次搭建并且写博客也有一年多了，那博客的域名已忘记，服务器估计早已经欠费回收，草稿随之而去。这样的事出现过10次对我来说都不是夸张，是精确[挖鼻屎]。

在一段新的半途而废旅程启动之前，先缅怀一下那些逝去的网站。

dijkst.github.io，是我之前参与的，唯一一个，还存活的博客，当时就已经看透自身的懒惰，找 Whirlwind 合写一个。那时的确保持了一段时间，两人写出了一些比较实用的文章，但终究敌不过本性，是我先不写了，Whirlwind 后来还偶尔有写。

再之前就是刚开始工作以及在大学的时候，冲动一来就注册一个 cnblog，csdn 志向满满地写两篇，也不知道那时写了什么鬼，估计都是C++基础之类的东西，当时就信仰这个...

时间再往前，高中的时候，到处找免费空间，10M，100M，那时也不知道ASP是什么，只要找到免费的静态空间就很开心了，往里面上传HTML文件，或者制作好的Flash动画。我还记得那时制作的那个Flash动画，是一个长条形流星掉进池塘却没有溅起水波的虚幻场景，现在看来和中老年热爱的表情样式并无太大差异。

我回头思考了一下，为啥那么多建站，写作经历都半途而废告终。我想除了自身因素，主要就两个原因：

一、建站的平台活得不够长，或者太容易被淘汰，难以一劳永逸。建站平台获得不够长很好理解，不少免费空间可能活不过2年，有的赚一笔就走，有的做不下去。一些远古时代著名的程序员博客网站，现在看，和几年前看，基本上没啥区别，几年前看可能觉得牛逼，现在则会觉得体验太差，不是他们变了，是时代变了。像Wordpress这样的建站工具。

二、独立建站和维护实在太太太麻烦了，Wordpress安装配置要好一会儿，选主题，安全设置都要花心思，偶尔出一个漏洞还要及时去补。之前的 Github Page 方式我只用过老款 Octopress 对于低频使用来说，记住使用方式也是一个巨大障碍，新电脑重新配置相当于重新学（对于懒人来说，什么都可以是借口）。

那么麻烦，为啥我还是很冲动地开始写了呢，因为以前同事 LoeiFy@github 弄了一个博客引擎 AcyOrt，基本上解决了上面的问题，基于估计可以活很久的 github 和 travis-ci ，只要在第一次使用时在自己电脑编辑两个文件，以后都无需用自己电脑敲命令，无需登录服务器，不想写了博客也一直存在，能联网就能写，想想都特美好。

新的博客，我不打算写太多关于技术细节的事情，主要记录一些有趣的想法，练习一下写作。
